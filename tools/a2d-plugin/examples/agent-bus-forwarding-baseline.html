4+1 视图

逻辑视图 logical

C-side(business_app)写消息到 agent-bus,经 ForwardingOutbox 持久化后由 ForwardingDispatchLoop 订阅投递,经 ForwardingDeliveryPort 抽象端口落到具体 transport(A2A HTTP adapter),消费方 agent-runtime 从 ForwardingInbox 收取。

评审风险
  • delivery 端口抽象与具体 transport(A2A)耦合度需持续守护,避免抽象泄漏
  • ForwardingRetryPolicy 的重投时机策略与 transport 实现解耦(separation of concerns)

开发视图 development

纯 Java 生产代码 + Postgres 持久化子包。ArchUnit 守护纯度:Spring/JDBC 圈进 persistence.jdbc 子包,业务内核保持框架无关。

允许依赖
  • agent-bus 内核 → Spring JDBC(仅 persistence.jdbc 子包)
  • ForwardingRetryPolicy → overflow-safe 指数退避(纯算子,无 I/O)
  • A2aForwardingDeliveryPort → A2A HTTP SDK
禁止依赖
  • agent-bus 内核不得直接 import Flyway(迁移归 persistence.jdbc/资源)
  • 业务内核不得依赖 System::currentTimeMillis(注入 EpochClock,ArchUnit 强制)
  • runtime 不得在 deliver 的非 lease 路径抛出未捕获 RuntimeException
评审风险
  • ArchUnit 纯度 trip-wire 范围随子包扩张需复核

进程视图 process

ForwardingDispatchLoop 以 TickSource 驱动 runOnce:claim 未投递记录 → 续约 lease → deliver → 按结果 ack/nack。无内嵌 scheduler,IdleStrategy 注入。

运行时流程
  • claimDue 取代 findRetryable:SELECT FOR UPDATE SKIP LOCKED 取一条 PENDING
  • 续约 DispatchLeasePolicy:deliver 前 renew,防止并发重复投递
  • deliver 成功 → markDelivered;deliver 失败分类 → RETRY/重投/SKIP
  • attemptCount 耗尽 → 进 DLQ,不留悬挂记录
失败 / 重试 / 取消路径
  • deliver 抛非 lease RuntimeException → 兜底 skipped,留 DISPATCHING 待重投(不丢消息)
  • lease 过期 → record 回 PENDING,由下一轮 claim 重新拾取
  • worker tick 抛异常 → try-catch 恢复 + DispatchTickResult.skipped 自洽,不让 loop 死
  • 自然 loop 永不触发续约 bug → 注入 EpochClock 修复(自然时钟难以测试)
评审风险
  • DispatchTickResult.skipped 路径的幂等性依赖 lease 闭环,跨实例重启边界需对账

物理视图 physical

持久化载体 Postgres;agent-bus 自有 Spring JDBC adapter;迁移 Flyway V1; 多租户用 RLS(行级安全)隔离。real-SQL 验证载体用 Zonky embedded-postgres。

部署影响
  • Postgres schema V1:ForwardingOutbox / ForwardingInbox / Lease 三表 + DDL CHECK
  • Flyway V1 migration 落 DDL + RLS 策略
  • RLS:tenant_id 带内字段(R-C.c),session 变量设租户上下文
  • adapter 不依赖具体载体生产可切:embedded-postgres(测)↔ 生产 PG ↔ Testcontainers
数据 / 租户 / 凭证 / 网络边界
  • 租户边界:RLS 行级隔离,无跨租户查询
  • 数据边界:outbox(sender)/inbox(receiver)分表,claim/lease 闭环不跨表
  • 凭证边界:PG 连接凭证由部署注入,agent-bus 不持有明文
  • 网络边界:A2A HTTP adapter 出站,需出站策略与超时配置
评审风险
  • embedded-postgres 验证 ≠ 生产载体验证,连接池/超时生产参数待定
